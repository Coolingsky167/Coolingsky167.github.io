泛型和类型安全容器

没有使用泛型的例子

apple和orange是有区别的，除了都是继承自Object之外没有任何共性。但是因为ArrayList保存的是Object，所以arraylist中既可以保存apple也可以保存orange，并且不论在编译期还是运行期都不会产生问题。但是当你从arrayList中取出对象的时候，得到的只是Object的引用，必须进行强制的类型转换（在调用id方法之前）。在运行时，就会出现类型转换错误java.lang.ClassCastException: pojo.orange cannot be cast to pojo.apple

使用泛型的例子

ArrayList<>内表示的是类型参数，指定了容器实例可以保存的类型，这样，就可以在编译期发现这个错误，这个时候，从容器中取出apple也不再需要类型转换

此外，ArrayList<apple>中可以存放apple子类

基本概念

Java容器类类库的目的是保存对象，分为两个不同的概念

**Collection: **一个独立元素的序列，这些元素都服从一条或者多条规则。List必须按照插入的顺序保存元素，Queue按照排队规则来确定对象产生的顺序（通常和他们被插入的顺序相同）

Map: 一组成对的“键值对”对象，允许你使用键值来查找值。ArrayList允许你使用数字来查找值，因此，从某种意义来说，他将数字和对象关联在了一起。映射表（Map）允许我们使用另一个对象来查找某个对象，因此被称为关联数组，或者“字典”

一般情况下，我们都是通过接口使用这些容器，但是有时候，因为一些额外的功能（比如LinkedList中包含List接口中没有的方法）就不能使用向上转型为更通用的接口

添加一组元素

Arrays.asList()

该方法是将数组转化成List集合的方法。

注意：

（1）该方法适用于对象型数据的数组（String、Integer…）

（2）该方法不建议使用于基本数据类型的数组（byte,short,int,long,float,double,boolean）

（3）该方法将数组与List列表链接起来：当更新其一个时，另一个自动更新

（4）不支持add()、remove()、clear()等方法

Collections.addAll()

该方法接收一个Collection对象，以及一个数组或者用逗号分隔的列表，将元素添加到Collection中

  • If you’re adding elements from an array, you can use Collections.addAll(col, arr)
    • Remember that varargs are also done using arrays
  • If you’re adding elements from a Collection, use col.addAll(otherCol)
    • Do NOT e.g. Collections.addAll(col, otherCol.toArray())
      • Such roundabout way is likely to be slower!
  • It’s not that one is supremely faster than the other
    • It’s about skipping unnecessary steps given the current situation

List

List允许它在被创建之后添加元素，移除元素，或者自我调整尺寸。

  • 基本的ArrayList中，长于随机访问元素，但是在List中间插入和移除元素时较慢
  • LinkedList，通过代价较低的在List中间进行的插入和删除操作，提供了优化的顺序访问。

迭代器

ListIterator

迭代器是一个对象，他的工作是遍历并选择序列中的对象，而客户端程序员不必知道或关心该序列底层的结构。

  • 使用iterator()要求容器返回一个Iterator，Iterator将准备好返回序列的第一个元素
  • 使用next()获得序列中的下一个元素
  • 使用hasNext()检查序列中是否还有元素
  • 使用remove()将迭代器新返回的元素删除

LinkedList

LinkedList和ArrayList一样实现了基本的List接口，但是它执行某些操作（在List中间插入和移除）时比ArrayList更加高效，但在随机访问操作方面逊色。同时LinkedList添加了可以使其作用于栈、队列的方法

Stack

LinkedList具有直接实现栈的所有方法，可以直接使用。

Java1.0的stack（util.Stack）继承自Vector，拥有比栈多的行为（不要去使用）
